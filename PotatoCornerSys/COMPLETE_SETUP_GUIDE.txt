## STEP 1: DATABASE SETUP
═══════════════════════════════════════════════════════════════

1. Open SQL Server Management Studio (SSMS)
2. Connect to your SQL Server instance
3. Open the file: CompleteDatabase_Setup.sql
4. Execute the script (F5 or click Execute)

This will:
✓ Create PotatoCornerDB database
✓ Create all 6 tables with proper relationships
✓ Add stock quantities to Products and ProductFlavors
✓ Insert admin account (admin / admin1290)
✓ Insert 2 sample users
✓ Set up all constraints and foreign keys

IMPORTANT: This script drops the existing database if it exists!
Make sure to backup any important data first.

## STEP 4: PASSWORD SECURITY (OPTIONAL)
═══════════════════════════════════════════════════════════════

Passwords are currently stored as plain text for easy testing.

To enable password hashing:
1. Run HashExistingPasswords.sql in SSMS
2. All passwords will be hashed with SHA256
3. Login will still work with original passwords

The system supports both plain text and hashed passwords
(backward compatibility built-in).

## DATABASE STRUCTURE
═══════════════════════════════════════════════════════════════

USERS Table:
- CustomerID (PK, Identity)
- UserName (Unique)
- Fullname
- Address
- Email (Unique)
- PhoneNumber (Unique, 11 digits)
- Password (Plain text or SHA256 hash)
- Points (Default: 0)
- MembershipLevel (Regular/Royalty/Admin)
- DateCreated

Membership Table:
- MembershipID (PK, Identity)
- CustomerID (FK to USERS)
- MembershipNumber (Unique)
- MembershipType (Default: Royalty)
- DateJoined
- ProfileImagePath (For future use)

Products Table (Sizes):
- ProductID (PK, Identity)
- ProductName (Unique)
- Price
- StockQuantity (Default: 50)
- Category (Default: Size)

ProductFlavors Table:
- FlavorID (PK, Identity)
- FlavorName (Unique)
- StockQuantity (Default: 50)

Orders Table:
- OrderID (PK, Identity)
- CustomerID (FK to USERS)
- CustomerName
- PhoneNumber (11 digits)
- Address
- TotalAmount
- OrderDate
- OrderStatus (Pending/Confirmed/Out for Delivery/Delivered/Picked Up/No Show/Cancelled)
- OrderType (Delivery/Walk-in)
- PickupLocation
- ScheduledPickupTime
- ActualPickupTime

OrderItems Table:
- OrderItemID (PK, Identity)
- OrderID (FK to Orders)
- ProductID (FK to Products)
- FlavorID (FK to ProductFlavors)
- Quantity
- Price

## STOCK MANAGEMENT
═══════════════════════════════════════════════════════════════

Initial Stock Levels:
- All sizes: 50 units each
- All flavors: 50 units each

Stock Deduction Rules:
- Size: 1:1 ratio (1 order = 1 unit deducted)
- Flavor: 1:10 ratio (1 order = 0.1 unit deducted)

Stock Refund Rules (No Show/Cancelled):
- Size: Full refund (1:1)
- Flavor: Full refund (1:10)

Update Stock:
- Use Update.aspx (Admin only)
- Enter new stock quantity
- Click Update button

## TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

Issue: Cannot login
Solution: Run ResetPasswordsToPlainText.sql to reset passwords

Issue: Stock not deducting
Solution: Check if Products and ProductFlavors have StockQuantity column

Issue: Name field is empty in Order.aspx
Solution: Make sure user is logged in (Session["Name"] exists)

Issue: Phone validation not working
Solution: Clear browser cache and refresh page
